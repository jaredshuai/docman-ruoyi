## Project Overview

`ruoyi-docman` 是 `docman-ruoyi` 聚合仓中的文档管理后端模块，负责项目、流程、节点上下文、插件执行、文档产物、归档和看板能力。模块依赖宿主后端运行，但其业务边界、数据模型和测试覆盖都相对独立。

## Quick Reference

- **Tech Stack:** Java 17, Maven, Spring Boot 3.5.9, MyBatis-Plus, Sa-Token, Warm-Flow, Redis, OSS/本地存储降级
- **Primary Entry Points:** `controller/*Controller.java`, `listener/DocmanNodeListener.java`, `job/*`
- **Architecture Pattern:** `controller -> application -> service/domain -> infrastructure/mapper -> plugin runtime`
- **Database:** `doc_project`, `doc_project_member`, `doc_process_config`, `doc_document_record`, `doc_archive_package`, `doc_node_context`, `doc_node_deadline`, `doc_plugin_execution_log`
- **Testing:** 69 个测试文件，覆盖 controller/application/service/domain/infrastructure/job/listener/plugin/sql

## Getting Started

### Prerequisites

- JDK 17
- Maven 3.9+
- Redis
- 数据库
- Warm-Flow 环境
- 宿主后端运行环境

### Setup

```bash
mvn -pl ruoyi-modules/ruoyi-docman -am clean compile
```

### Run Locally

```bash
# 先编译模块，再通过宿主后端启动并访问 /docman/**
mvn -pl ruoyi-modules/ruoyi-docman -am clean compile
```

### Run Tests

```bash
mvn -pl ruoyi-modules/ruoyi-docman -am -DskipTests=false test
```
